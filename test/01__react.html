React

1. 관리자 사이트를 통한 리액트 앱설정 및 데이터 모니터링 (제휴사업팀: 양희영과장, 박지혜 사원 / 헬스케어서비스팀)

 - 본 내용은 별도의 관리자 매뉴얼 문서로 제공함

 - 운영관리자 사이트 매뉴얼

 - 모니터링 관리자 사이트 매뉴얼

 - 기존 앱내 문의에 대한 케이스별 답변 안내


2. 추가개발 관련 (ICT팀: 김정우팀장 / SBD팀: 유흔호부장 / 제휴사업팀: 박지혜 사원)

 - 추가개발 기획서 http://sb.aimmed.com:8102/react_v2

 - 추가개발에 대한 프로젝트 관리는 ICT팀 김정우팀장 담당


3. 산출물

 - 1차 산출물은 별도의 USB 메모리로 ICT팀에 전달함

 - 추가개발 후 산출물 최종 업데이트 요청


4. 프로젝트 진행

 - 프로젝트 진행을 위한 도구는 https://trello.com를 이용함 (업체에서 초대를 통해 이용)


5. 앱업데이트 관련

 - 앱 업데이트는 업체에서 진행하며, Google, Apple 개발자 계정으로 진행함 (계정은 ICT팀에서 보유)


6. 앱테스트 버전 관련 (ICT팀: 김정우팀장 / SBD팀: 유흔호부장 / 제휴사업팀: 박지혜 사원)

 - 앱 개발버전이 업데이트가 되면 앱스토어 등록전에 테스트를 위하여 회사 테스트폰 Note3, G2에 설치 가능 (Beta앱 확인)


7. 모니터링 관리자(Django Admin) 기본 코드 안내 (ICT팀: 김정우팀장)

 - API 서버의 var/react-django-admin 폴더의 app폴더와, project폴더를 확인

 - DB의 Field 정의는 app폴더의 models.py에 있음

 - Admin 페이지의 데이터별 기능에 대한 Code는 app폴더의 admin.py에 있음

 - models.py에 정의된 model class에 대한 기능을 admin.py에 class로 작성되어 있음

 - project폴더의 settings.py에 DB설정등에 대한 정보가 있음


8. 모니터링 관리자의 프로세스 관련 (ICT팀: 김정우팀장)

 - $ gunicorn project.wsgi:application --bind=127.0.0.1:8001 --daemon --reload / Django Admin 데몬 실행

 - $ ps -ef | grep gunicorn / Django Admin 프로세스 확인

 - $ ps kill {프로세스 번호} / Django Admin 프로세스 Kill
